## Community Tips

### Multi-Agent Setup Best Practices

**1. Agent Routing Configuration**
```json
{
  "agents": {
    "list": [
      {
        "id": "personal",
        "workspace": "~/.openclaw/workspace-personal",
        "model": "anthropic/claude-sonnet-4-5"
      },
      {
        "id": "work", 
        "workspace": "~/.openclaw/workspace-work",
        "model": "anthropic/claude-opus-4-6"
      }
    ]
  },
  "bindings": [
    {"agentId": "personal", "match": {"channel": "whatsapp"}},
    {"agentId": "work", "match": {"channel": "telegram"}}
  ]
}
```

**2. Security Hardening**
- Always sandbox untrusted agents: `sandbox: { mode: "all", scope: "agent" }`
- Use tool allowlists for restricted agents: `tools: { allow: ["read"], deny: ["exec"] }`
- Separate auth profiles per agent to prevent credential leakage
- Enable VirusTotal scanning for ClawHub skills before installation

**3. Production Deployment Patterns**
- **DigitalOcean App Platform**: Elastic scaling + Tailscale private access
- **Railway**: Single-click deploys with environment isolation
- **Self-hosted**: Docker containers with volume mounts for persistence

**4. Channel Management**
- Use `accountId` to separate multiple WhatsApp/Telegram accounts
- Route specific DMs to specialized agents via peer matching
- Set `mentionPatterns` for group chat agent targeting
- Enable `dmPolicy: "allowlist"` for controlled access

### Configuration Optimizations

**Model Selection by Channel**
- WhatsApp → Fast model (Sonnet) for daily chat
- Telegram → Thinking model (Opus) for deep work
- Discord → Collaborative model for team coordination

**Memory Management**
- Use persistent sessions for long-running conversations
- Enable memory syncing across agent instances
- Implement conversation archiving for performance

## Security Recommendations

### Immediate Actions Required

1. **Skill Vetting Pipeline**
   - Deploy Cisco's Skill Scanner tool
   - Implement VirusTotal integration
   - Manual code review for critical skills
   - Maintain allowlist of trusted skill authors

2. **Isolation & Sandboxing**
   - Enable Docker sandbox for all external skills
   - Separate credential stores per agent
   - Implement tool allowlists for restricted operations
   - Monitor file system access and network traffic

3. **Access Controls**
   - Review and minimize tool permissions
   - Enable audit logging for all agent actions
   - Implement rate limiting for API calls
   - Add emergency kill switches for runaway agents

### Long-term Security Strategy

- Contribute to OpenClaw security initiatives
- Develop internal skill testing framework  
- Share security findings with community
- Maintain private skill repository for sensitive operations
