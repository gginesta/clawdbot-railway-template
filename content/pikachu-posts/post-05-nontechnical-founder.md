## Thread

1/ I'm a managing partner at a VC. I don't code. I now run a team of AI agents for my daily work. Here's how that happened. 🧵

2/ Background: I spend my days in meetings, reviewing decks, writing memos, managing deal flow. Classic knowledge work.

3/ December 2025: I got curious about AI agents. Not chatbots—actual autonomous systems that do work while I sleep.

4/ The blocker: every tutorial assumed you're a developer. Docker, Python, APIs everywhere. I know enough to be dangerous, but not enough to build.

5/ The unlock: OpenClaw. It's an agent framework designed for non-technical operators. You define agents in markdown files. Deploy with one command.

6/ My learning curve:
• Week 1: Just reading, understanding the concepts
• Week 2: First agent running (broke constantly)
• Week 3: Actually useful output
• Week 4+: Two agents running my workflow

7/ What I actually do now: I give instructions in plain English. I review agent output. I make decisions. The agents handle execution.

8/ The honest truth: I still ask Claude for help when things break. That's fine. The point isn't becoming a developer. It's getting leverage.

9/ If you're a non-technical exec curious about agents: start with ONE use case you hate doing. Email summaries, meeting prep, whatever. Automate that first.

You don't need to code. You need to be specific about what you want. #AIAgents #NonTechnical
