# Agent Performance Reviews

**Cadence:** Monthly (first Monday of month)
**Reviewer:** Molty compiles → Guillermo reviews → shared with agent
**Time:** ~15 min to compile per agent

---

## Framework

### 1. Scorecard (Quantitative)

| Metric | Source | Target |
|--------|--------|--------|
| Tasks completed | MC done count | Track trend |
| Tasks assigned | MC assigned count | — |
| Completion rate | completed / assigned | >80% |
| Regressions caused | REGRESSIONS.md entries | 0 |
| Communication failures | Closed without review, stale data, silent failures | 0 |

### 2. Highlights + Lowlights (Qualitative)

- **Best work this month:** One standout deliverable or behavior
- **Worst failure this month:** One thing that broke trust or caused problems
- **Pattern:** Improving / Flat / Regressing (compared to last month)

### 3. Focus for Next Month

- **Get better at:** One specific skill or behavior to improve
- **Stop doing:** One specific pattern to eliminate

---

## Process

1. **Day 1 of month:** Molty pulls MC data for each agent (prev month)
2. **Day 1-2:** Molty writes draft reviews
3. **Day 2-3:** Guillermo reviews, adds comments
4. **Day 3-5:** Share with each agent, discuss if needed

---

## Philosophy

- Reviews are for learning, not gatekeeping
- Trust + coaching > restrictions
- Be specific — "communicate better" is useless, "tell Guillermo when task is done" is actionable
- Trends matter more than single incidents
