## When to Use Perplexity Sonar Models
| Model | Alias | Best For |
|-------|-------|----------|
| `perplexity/sonar-pro-search` | `sonar` | **Fast web search** â€” real-time facts, current events, quick lookups. Replaces Brave for accuracy. Use when sub-agents need fresh web data (news, prices, company info, recent announcements). |
| `perplexity/sonar-deep-research` | `deep-research` | **Deep synthesis** â€” multi-source research reports, competitive analysis, thorough investigations. Use for /last30days-style tasks, market research, due diligence, "become an expert on X". Takes longer but far more thorough. |

**When to use which:**
- Quick factual lookup â†’ `sonar`
- "What's happening with X lately?" â†’ `sonar`
- "Research topic X thoroughly" â†’ `deep-research`
- "What's new monitoring" recurring checks â†’ `sonar` (fast, cheap)
- One-off deep dive reports â†’ `deep-research`
- Simple web scraping / specific URL â†’ still use `web_fetch` (free, no API cost)
- General knowledge questions â†’ don't need either (use main model's training data)

**Cost note:** Both route through OpenRouter, so they consume OpenRouter credits. Use judiciously for sub-agents â€” don't default every search to Sonar when Brave suffices for simple queries.
