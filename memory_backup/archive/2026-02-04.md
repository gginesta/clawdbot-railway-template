## Context Overflow Crashes (04:05 UTC)

**Problem:** Crashed 3 times with "Context overflow: prompt too large for model" when looking at logs and retrieving information.

**Root cause:** Reading large files without limits. Session JSONL files can be 15MB+. Even one careless `Read` can blow up context.

**Analysis:**
- Largest session file: 15MB, 1923 lines
- MEMORY.md + all daily files: ~1159 lines (manageable)
- Problem is session/log files, not memory files

**Operational Rules (MUST FOLLOW):**

1. **Never read entire log/session files**
   - Use `tail -100 file.log` or `head -100`
   - Use `limit` param on Read tool: `Read(file, limit=100)`
   - For JSONL: `tail -50 file.jsonl | jq .`

2. **Targeted searches over bulk reads**
   - `grep -A5 -B5 "error" file.log` (context around matches)
   - `jq 'select(.type=="error")' file.jsonl`

3. **Memory search with limits**
   - Always set `maxResults: 5` (or less)
   - Use `minScore: 0.5` to filter noise

4. **One chunk at a time**
   - Don't stack multiple large file reads
   - Process, summarize, then get more if needed

5. **Check file size first**
   - `wc -l` or `ls -lh` before reading unknown files
   - If >500 lines, use surgical extraction
