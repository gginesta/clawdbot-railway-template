## 22:12 HKT — Webchat Fix Applied

### Root Cause Found
Logs showed:
```
[ws] Proxy headers detected from untrusted address. Connection will not be treated as local.
[ws] closed before connect
```

**Problem:** Railway's internal proxies (100.64.0.x CGNAT range) weren't in `gateway.trustedProxies`. Only had `127.0.0.1`.

### Fix Applied
```json
"gateway": {
  "trustedProxies": ["127.0.0.1", "100.64.0.0/10"]
}
```

Config patched via `gateway config.patch`. Restart triggered.

### Lesson Learned
**REG-019:** On Railway deployments, always include `100.64.0.0/10` in `gateway.trustedProxies` — this is Railway's internal CGNAT range used by their proxy infrastructure.
