## Update 20:27 HKT — Full Session Findings

### Smoking Gun (from build logs)
All failed deploys show the SAME healthcheck pattern in **build logs** (not runtime logs):
```
Starting Healthcheck
Path: /api/health
Attempt #1 failed with service unavailable. Continuing to retry for 58s
Attempt #2-6: service unavailable
1/1 replicas never became healthy!
```
**"Service unavailable" = 503** — Railway's reverse proxy receives the probe but can't route to the container. The server IS listening, but Railway's internal networking hasn't wired the port mapping by the time probing starts.

### Why cached image redeploys work
Redeployed builds reuse existing network configuration. Fresh builds get new containers that need to be wired into Railway's routing mesh — and that wiring isn't completing before the 60s healthcheck timeout expires.
