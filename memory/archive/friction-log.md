# Friction Log

**Purpose:** Track contradictory instructions rather than silently resolving them. Surface at next break point.

**When to use:** Two instructions conflict (AGENTS.md vs direct Guillermo, past decision vs new request, agent domain vs Guillermo override).

**Process:**
1. Log immediately when detected
2. Note how I resolved it
3. Mark "Surfaced: Yes/No"
4. At next interaction, surface unresolved items
5. Guillermo decides which is canonical → update source document

```markdown
- **Resolution:** <how I handled it>

- **Decision:** [Guillermo's canonical answer once resolved]
```
