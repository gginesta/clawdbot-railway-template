## LESSON #50: `railway run` and `railway shell` are ALWAYS LOCAL
- They inject Railway env vars into a local shell — they do NOT SSH into the remote container
- `railway run --service X cat /file` reads YOUR LOCAL filesystem, not the remote service
- The ONLY way to access a remote Railway container's filesystem is via the setup API or the service's own endpoints (webchat, webhooks)
- This means earlier "checks" of Leonardo's files via `railway run` were actually reading LOCAL files — they only looked correct by coincidence (identity swap bug)
- For Yoda: setup API is the only way to write config, and it requires browser for the initial wizard
