**Export bug discovered:** `gws auth export` fails to find encrypted credentials even when they exist. Workaround: manually copy these files from authenticated machine:
- `.encryption_key`
- `accounts.json`
- `credentials.<base64-email>.enc`

Then base64-decode the .enc file on receiving end:
```powershell
[Convert]::ToBase64String([IO.File]::ReadAllBytes("path\to\file.enc"))
```

### 2. Webchat → Telegram Duplicate Messages — ROOT CAUSE FOUND

**Issue:** Messages sent via webchat were also appearing on Telegram.

**Root cause:** Session channel inheritance. When you message on Telegram first, the session's "channel" is set to Telegram. When you switch to webchat:
- You're viewing the SAME session
- Replies route to BOTH: webchat (current view) AND Telegram (session's channel)

**Solution:** Start session from webchat first, OR run `/reset` in webchat to create a new session with webchat as the channel.

**Session metadata when this happens:**
```json
{
  "channel": "telegram",      // ← replies go here
  "provider": "webchat",      // ← message came from here
  "surface": "webchat"
}
```

**Permanent fix options:**
1. `session.dmScope: "per-channel-peer"` — separate sessions per channel (breaks continuity)
2. User starts session from preferred channel first
3. Mute Telegram when using webchat (workaround)
