## Evening Session: Agent-to-Agent Communication Solution

### Problem Discovered
Telegram bots cannot receive messages from other bots — this is a **Telegram API limitation**, not an OpenClaw issue.
- Discord also had issues earlier
- Needed a reliable alternative

### Solution: Webhook-Based Direct Communication ✅

Bypasses all messaging platform limitations by using direct HTTP webhooks between OpenClaw instances.

**How it works:**
```
Molty (ggvmolt.up.railway.app)
    ↓ POST /hooks/agent
Raphael (ggv-raphael.up.railway.app)
    ↓ POST /hooks/agent  
Molty receives response
```

## Evening Session: Webhooks Working! 🎉

### Railway Template Bug Fixed
**Problem:** Webhooks returning 400 errors — `express.json()` middleware was being applied globally, which broke the OpenClaw-proxied routes.

**Fix:** Modified template to only apply `express.json()` to `/setup` routes.
- PR merged to our fork: https://github.com/gginesta/clawdbot-railway-template
- Both instances redeployed with fix

### Critical Discovery: sessionKey Required!
Agent webhooks were "working" (200 response) but messages weren't appearing in Raphael's main session.

**Root cause:** Without `sessionKey`, messages went to a fresh isolated session instead of the main session.

**Working payload format:**
```json
{
  "message": "Your message here",
  "sessionKey": "agent:main:main",
  "wakeMode": "now"
}
```
