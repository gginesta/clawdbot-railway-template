## Fixes (Relevant to Our Setup)

### Cron System (Critical for Us)
- Prevent duplicate fires when multiple jobs trigger simultaneously
- Re-arm timer during active job execution
- Isolate schedule errors (one bad job won't break others)
- Pass agentId correctly for main-session jobs
- Prevent one-shot jobs from re-firing on restart

**Impact:** Our daily standup, backup, and update cron jobs are now more reliable.

### Telegram
- Handle no-text messages in model picker
- Surface REACTION_INVALID as non-fatal warning (no more crashes)
- Add retry logic to health probe
- Transcribe audio before mention check in groups with requireMention

**Impact:** Molty's Telegram integration more stable, especially in groups.

### WhatsApp
- Markdown bold/strikethrough conversion
- Default MIME type for voice messages
- Media-only sends supported

**Impact:** If we add WhatsApp for any agent, better formatting support.

### Memory/QMD
- Treat plain-text no-results as empty
- Harden no-results parsing
- Guard local media reads

**Impact:** Memory search more reliable, fewer edge case failures.

### Subagents
- Report subagent timeout as "timed out" instead of "completed successfully"
- Exclude rate limit errors from context overflow classification
- Enable tool call ID sanitization for Anthropic provider
- Scope process/exec tools to sessionKey for isolation

**Impact:** Better error reporting when spawning sub-agents for research tasks.

### Discord
- Default standalone threads to public type
- Detect control commands when message starts with @mention

**Impact:** Raphael and Leonardo's Discord bots handle threads correctly.

### Slack
- Default replyToMode from "off" to "all"

**Impact:** If we add Slack, replies work better by default.

### Gateway
- Auto-generate token during `gateway install` (no more restart loops)
- Drain active turns before restart (no message loss)
- Handle async EPIPE on stdout/stderr during shutdown

**Impact:** Smoother restarts for Railway deployments.
