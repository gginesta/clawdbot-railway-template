## Standup Script Bug Fix — 2026-03-03 (~17:35 HKT)

### Bug
`daily_standup.py` crashed at 5:00 PM cron with:
`UnboundLocalError: cannot access local variable 'today_str' where it is not associated with a value`

### Root Cause
Python variable shadowing. In `main()`:
- Line 1260 assigned `today_str = today.strftime(...)` (local variable)
- Python therefore treated `today_str` as local for the ENTIRE function scope
- So the earlier call `today = today_str()` at line 1170 failed (local var not yet assigned)

Secondary bug: `today` is a string (from `today_str()`), but lines 1242 + 1260 called `.strftime()` on it.

### Fix (git commit 9a246735)
1. Renamed local `today_str` variable → `today_date` (avoids shadowing the function)
2. Changed `today.strftime('%Y-%m-%d')` → `today` (already a string, no need to reformat)
3. Changed `today_date = today.strftime(...)` → `today_date = today`

### Result After Fix
Script runs cleanly. Page created. Telegram summary generated.
Standup page for 2026-03-03: `31839dd6-9afd-81b5-ad05-d6b4f62af269`

### Lesson
Variable name collision in Python: assigning to a name anywhere in a function makes Python treat it as local for the entire function scope. Never use a variable name that matches a module-level function name inside a function that also calls that function.
